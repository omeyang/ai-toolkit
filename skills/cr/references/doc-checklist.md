# 文档检查清单

当审查范围包含文档文件（`.md`、`README`、`CHANGELOG`、`doc.go`、docstring 等）时，应用此清单。

---

## 通用文档检查

### A 级（阻塞）

- **[A-DOC-01] 技术错误** — 文档中的代码示例、命令、配置包含语法错误或已过时的 API 调用。
- **[A-DOC-02] 安全误导** — 文档示例包含不安全的做法（明文密码、禁用 TLS、忽略错误）且未标注警告。
- **[A-DOC-03] 版本不一致** — 文档描述的版本号、依赖版本与实际代码/配置不匹配。

### B 级（重要）

- **[B-DOC-01] README 不完整** — 缺少以下关键章节之一：项目简介、安装方式、基础用法、许可证。
- **[B-DOC-02] 代码示例不可运行** — 示例代码缺少必要的 import、变量声明，无法直接运行。
- **[B-DOC-03] 缺少变更记录** — 公开 API 变更但未更新 CHANGELOG 或 Release Notes。
- **[B-DOC-04] 链接失效** — 文档中的链接指向不存在的文件或页面。
- **[B-DOC-05] 配置说明缺失** — 引入了新的配置项/环境变量但文档中未说明用途和默认值。

### C 级（建议）

- **[C-DOC-01] 拼写和语法** — 明显的拼写错误或语法问题，影响专业性。
- **[C-DOC-02] 格式不一致** — 标题层级、列表风格、代码块标记等格式不统一。
- **[C-DOC-03] 缺少示意图** — 复杂架构或流程缺少可视化说明。

---

## Go 文档检查

### B 级（重要）

- **[B-DOC-GO-01] 导出标识符无文档** — 公开的类型、函数、方法、常量缺少 godoc 格式的注释（以标识符名开头的句子）。
- **[B-DOC-GO-02] 缺少 doc.go** — 包含多个文件的包缺少 `doc.go` 文件来说明包的整体用途。
- **[B-DOC-GO-03] 缺少 Example 函数** — 核心公开 API 缺少 `Example_*` 测试函数，无法在 godoc 中展示用法。

### C 级（建议）

- **[C-DOC-GO-01] 包注释过短** — 包注释只有一行，对于复杂包应补充使用说明和设计决策。
- **[C-DOC-GO-02] Deprecated 未标注替代** — 标注了 `Deprecated` 但未说明替代方案。

---

## Python 文档检查

### B 级（重要）

- **[B-DOC-PY-01] 公开函数无 docstring** — 公开函数/方法（非 `_` 开头）缺少 docstring。
- **[B-DOC-PY-02] docstring 格式不一致** — 项目中混用 Google、NumPy、reStructuredText 等不同 docstring 格式。
- **[B-DOC-PY-03] 类型注解与文档冲突** — docstring 中描述的参数类型与类型注解不一致。
- **[B-DOC-PY-04] `__all__` 缺失** — 包含多个公开函数的模块缺少 `__all__` 来明确公开 API。

### C 级（建议）

- **[C-DOC-PY-01] 模块级 docstring 缺失** — 模块（.py 文件）缺少文件级 docstring 说明模块用途。
- **[C-DOC-PY-02] 参数说明不完整** — docstring 中缺少部分参数的说明或返回值描述。

---

## Shell 文档检查

### B 级（重要）

- **[B-DOC-SH-01] 脚本头注释缺失** — 脚本缺少头部注释说明用途、用法和依赖。
- **[B-DOC-SH-02] 函数说明缺失** — 定义了函数但缺少注释说明参数和返回值。
- **[B-DOC-SH-03] 环境变量未文档化** — 脚本依赖环境变量但未在头部或 README 中列出。
- **[B-DOC-SH-04] 退出码未文档化** — 脚本使用了非零退出码但未说明各退出码的含义。

### C 级（建议）

- **[C-DOC-SH-01] 用法示例缺失** — 脚本缺少 `--help` 或 `-h` 选项，或帮助信息不包含使用示例。
- **[C-DOC-SH-02] 依赖工具未列出** — 脚本依赖外部工具（jq、curl、awk 等）但未在文档中列出。
